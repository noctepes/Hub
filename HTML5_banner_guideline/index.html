Tại sao quy trình này hiệu quả?

Quy trình này được thiết kế để tối ưu hóa việc sử dụng các công cụ chuyên biệt cho từng giai đoạn. Photoshop xuất sắc trong việc xử lý layout và tạo assets chất lượng cao, trong khi Adobe Animate CC được tối ưu hóa cho việc tạo animation HTML5 và export code sạch. Việc tách biệt các giai đoạn giúp kiểm soát chất lượng tốt hơn và dễ dàng thay đổi khi cần thiết.

Lưu ý quan trọng: Luôn backup các file gốc ở mỗi giai đoạn. Điều này giúp bạn có thể quay lại và chỉnh sửa khi khách hàng yêu cầu thay đổi mà không phải làm lại từ đầu.

Bước 1: Nhận Brief và Working File

1

Thu thập thông tin từ khách hàng

Bất kể khách hàng cung cấp file định dạng gì (AI, PSD, JPG, PNG, XD, inDesign, PDF, hay thậm chí là sketch trên giấy), nhiệm vụ của bạn là hiểu rõ ý tưởng và chuyển đổi tất cả thành format phù hợp để làm việc. Giai đoạn này rất quan trọng vì nó quyết định chất lượng của sản phẩm cuối cùng.

Thông tin cần thu thập
  • Kích thước banner (320x100, 728x90, 300x250, etc.)
  • Thời gian animation (thường 15-30 giây)
  • File size limit (thường < 150KB)
  • Click URL và tracking requirements
  • Brand guidelines và fonts
  • Storyboard hoặc mô tả animation
Lưu ý về chất lượng file: Nếu khách cung cấp file có độ phân giải thấp, hãy yêu cầu file gốc có chất lượng cao hơn hoặc vector files để đảm bảo chất lượng banner cuối cùng.

Bước 2: Tạo Layout trong Photoshop

2

Chuyển đổi tất cả file về Photoshop

Photoshop là công cụ tốt nhất để xử lý và chuẩn bị assets cho web. Ngay cả khi bạn nhận được file Illustrator, việc chuyển sang Photoshop giúp kiểm soát tốt hơn các yếu tố như độ phân giải, color profile, và export quality.

Cài đặt document Photoshop:
- Width: Theo yêu cầu banner (ví dụ: 320px)
- Height: Theo yêu cầu banner (ví dụ: 100px)
- Resolution: 72 PPI
- Color Mode: RGB Color/sRGB Color
- Background: PNG/JPG
3

Tổ chức layers theo frame/storyboard

Đây là bước quan trọng nhất trong việc chuẩn bị assets. Việc tổ chức layers một cách có hệ thống sẽ giúp bạn dễ dàng export và import vào Animate sau này. Hãy nghĩ về animation như một câu chuyện với nhiều frame khác nhau.

Chiến lược đặt tên layers
Cách đặt tên hiệu quả:
- Frame01_background
- Frame01_logo
- Frame01_text_headline
- Frame02_product_image
- Frame02_text_description
- Frame03_cta_button

Hoặc theo element:
- bg_static
- logo_animated
- text_headline_01
- text_headline_02
- button_cta
4

Export assets sử dụng Generate Plugin

Photoshop Generator là công cụ mạnh mẽ cho việc export assets tự động. Bằng cách đặt tên layers một cách thông minh, bạn có thể export hàng loạt assets với chất lượng và format phù hợp chỉ trong một lần click. Generator mặc định không kích hoạt, chọn setting → plugin → check enable generator → khởi động photoshop để kích hoạt generator.

Cách sử dụng Generator:
1. File → Automate → Generators Plugins → Image Assets
2. Đặt tên layer với extension:
   - logo.png
   - background.jpg
   - button.png
   - icon.png
3. Assets sẽ tự động export vào thư mục [filename]-assets
4. Chỉnh sửa chất lượng bằng cách thêm suffix
   - background.jpg5
   - background.jpg50%
Thêm kích thước hình ảnh đầu ra mong muốn (tương đối hoặc theo các định dạng được hỗ trợ: px, in, cm và mm) làm tiền tố cho tên asset. Photoshop sẽ tự động điều chỉnh tỷ lệ hình ảnh cho phù hợp. Ví dụ:
   - 200% logo.png
   - 300 x 200 Rounded_rectangle_3.jpg
   - 10in x 200mm Rounded_rectangle_3.jpg
Lưu ý: Nhớ thêm một ký tự khoảng trắng giữa tiền tố và tên asset. Nếu bạn chỉ định kích thước tính bằng pixel, bạn có thể bỏ qua đơn vị. Ví dụ: 300x200.
Pro tip: Sử dụng Smart Objects để giữ chất lượng vector khi resize. Convert text layers thành Smart Objects trước khi export để đảm bảo text không bị pixelated.

Bước 3: Tạo Animation trong Adobe Animate CC

5

Setup project trong Adobe Animate

Adobe Animate CC được tối ưu hóa đặc biệt cho HTML5 canvas animation. Việc setup đúng từ đầu sẽ giúp export code sạch và tối ưu performance.

Cài đặt Document:
- Platform Type: HTML5 Canvas
- Width/Height: Theo yêu cầu banner
- FPS: 24-30
- Background: Transparent
- Ruler units: Pixels
- Stage color: Theo design
6

Import assets và tổ chức Library

Việc import và tổ chức assets một cách khoa học sẽ giúp bạn làm việc hiệu quả và dễ dàng maintain project sau này. Animate CC có system Library rất mạnh mẽ để quản lý assets.

Quy tắc tổ chức Library
Cấu trúc folder Library:
📁 Images
  📁 Backgrounds
  📁 Products
  📁 Icons
📁 Text Elements
📁 Buttons
📁 Symbols
  📁 Animated
  📁 Static

Tips tối ưu:
- Convert bitmap thành Symbol để reuse
- Sử dụng compression settings phù hợp
- Group related elements
7

Tạo timeline animation

Timeline là trái tim của animation. Hiểu rõ cách sử dụng keyframes, tweens, và easing sẽ giúp tạo ra animations mượt mà và chuyên nghiệp. Adobe Animate cung cấp nhiều loại tween khác nhau để tạo hiệu ứng đa dạng.

Các loại animation phổ biến trong banner:

  • Fade In/Out: Hiện/ẩn elements
  • Slide/Pan: Di chuyển elements
  • Scale: Phóng to/thu nhỏ
  • Rotation: Xoay elements
  • Color Tint: Thay đổi màu sắc
  • Mask: Hiệu ứng reveal
Best practices cho Timeline:
- Sử dụng layer names có ý nghĩa
- Tạo guide layers cho timing reference
- Sử dụng frame labels để mark các scene
- Kiểm tra animation ở nhiều speed khác nhau
- Optimize keyframe usage
Lưu ý về performance: Tránh animation quá nhiều elements cùng lúc. Điều này có thể làm giảm performance trên thiết bị yếu. Ưu tiên transform properties (position, scale, rotation) thay vì filter effects.

Bước 4: Export và Optimize HTML5

8

Export HTML5 từ Adobe Animate

Việc export đúng cách từ Animate sẽ tạo ra code base sạch và tối ưu. Adobe Animate tự động tạo ra HTML, CSS, và JavaScript cần thiết cho banner, nhưng vẫn cần fine-tuning để đạt performance tốt nhất.

Export Settings:
File → Publish Settings → HTML5 Canvas

Basic:
- Output name: banner (sẽ tạo banner.html)
- Include hidden layers: OFF

Advanced:
- Compress JavaScript: ON
- Include preloader: ON (nếu cần)
- Host libraries externally: OFF
- Make responsive: Tùy requirement
9

Loại bỏ code thừa và optimize

Adobe Animate thường generate code với nhiều features không cần thiết cho banner đơn giản. Việc clean up code sẽ giảm đáng kể file size và tăng loading speed.

Các bước optimize code
1. Trong file HTML:
- Xóa các meta tags không cần thiết
- Minify HTML nếu có thể
- Remove unused stylesheets

2. Trong file JavaScript:
- Remove unused variables
- Combine similar functions
- Minify JavaScript code

3. Assets optimization:
- Compress images (TinyPNG, ImageOptim)
- Convert to WebP nếu support
- Remove unused assets
10

Đổi tên file và thêm Google Ad specs

Để banner hoạt động đúng trên các ad networks, đặc biệt là Google Ads, cần thêm các meta tags và specifications theo chuẩn. Điều này đảm bảo banner được hiển thị và track đúng cách.

Thêm vào <head> section:

<!-- Google Ad Specs -->
<meta name="ad.size" content="width=320,height=100">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<meta name="format-detection" content="telephone=no">

<!-- Clicktag Support -->
<script>
var clickTag = "";
</script>
Thêm click handler:

function bannerClick() {
  if (clickTag) {
    window.open(clickTag, '_blank');
  }
}

// Thêm vào canvas click event
canvas.addEventListener('click', bannerClick);
File naming convention:
- index.html (thay vì banner.html)
- index.js (file JavaScript chính)
- t1.png, t2.png, bg1.jpg, bg2.jpg.... (assets)
- Tất cả files nên lowercase, không dấu cách
Kiểm tra file size: Hầu hết ad networks có giới hạn file size (thường < 150KB cho tổng banner). Sử dụng tools như Google PageSpeed Insights để kiểm tra và optimize further nếu cần.

Performance Optimization

Techniques để tăng tốc banner:
